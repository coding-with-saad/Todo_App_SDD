# Frontend Skill – Pages, Components, Layout, Styling

## Instructions

1. **Page Structure**
   - Create pages for each main feature
   - Use clear layout hierarchy (header, content, footer)
   - Keep page logic minimal and delegate UI to components

2. **Component Design**
   - Build reusable UI components
   - Keep components small and focused
   - Pass data through props and avoid global state when possible

3. **Layout System**
   - Create a consistent layout wrapper
   - Maintain spacing, alignment, and responsive structure
   - Separate layout components from content components

4. **Styling**
   - Use consistent styling approach (CSS modules, Tailwind, or styled components)
   - Apply responsive design principles
   - Keep styles modular and maintainable

## Best Practices

- Follow a component-based architecture
- Keep UI components reusable
- Use mobile-first responsive design
- Avoid large monolithic pages
- Maintain consistent spacing, typography, and color usage
